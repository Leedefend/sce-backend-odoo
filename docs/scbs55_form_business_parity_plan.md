# SCBS55 表单与业务办理专题计划

## 目标

围绕“用户核对菜单”的 55 个用户可见入口，系统性解决新系统表单与老系统差距大、业务不能正常办理的问题。验收对象不是单个字段，而是用户实际办理链路：

- 列表进入表单正常
- 表单字段、分组、标签、必填、只读与老系统用户可见面一致
- 保存、新建、编辑、删除策略符合业务权限
- 表头按钮、状态按钮、审批/确认/退回等办理动作可用
- 明细行、附件、关联字段、金额税额等关键字段可录入和回显
- 历史迁移数据打开不报错，新建数据能走通当前业务流程

已从 dev 环境 `sc_demo` 抽取 55 个用户核对菜单的新系统 form 契约：

## 推进顺序

### P0：通用阻断修复

优先修复会导致所有表单不可办理或大面积不可用的问题：

- 表单保存失败
- 关联字段无法选择或显示错位
- 必填字段缺省导致无法提交
- object/action 按钮没有映射到 `execute_button`
- 状态流转按钮不可见或点击失败
- 附件上传/下载/显示异常
- one2many/many2many 明细行无法新增、编辑、删除

### P1：高频业务办理链路

优先验收用户最可能马上使用的业务：

- 支付申请
- 报销申请
- 到款确认表
- 预缴税款
- 进项发票登记
- 销项开票申请
- 施工合同
- 供应商/合作单位、往来单位
- 扣款单、扣款实缴登记
- 资金日报表

### P2：迁移数据回显与历史单据

对旧库回填数据做打开与只读/可编辑边界验收：

- 历史状态是否正确显示
- 历史附件是否可读
- 历史明细是否完整
- 项目、单位、人员、账户等关联字段是否回填正确
- 金额、税额、类型、日期是否无错位

### P3：报表与统计入口

报表类入口以“查询可用、字段口径正确、总数/合计不错”为主：

- 成本统计表（综合）
- 投标保证金报表
- 发票成本进度报表
- 发票分析报表
- 项目经营统计表
- 应收应付报表
- 账户收支统计表

## 完成标准

一个菜单达到“可交付”必须满足：

- 新老系统用户可见字段无重大缺口
- 至少 1 条历史迁移数据打开正常
- 至少 1 条当前业务数据可新建或按权限办理
- 表单主要按钮可见且可执行
- 关键金额/税额/日期/类型/项目/单位字段无错位
- 浏览器验收截图与 JSON 报告已归档
