# 账户收支统计表承载 Batch-S

## 批次定位

- Layer Target：Domain Projection

- 旧报表：账户收支统计表
- 旧过程：`Report_SP_USP_Select_ZHSZTJB_GS_Tree`

## 已承载内容

本批新增只读投影模型 `sc.account.income.expense.summary`，用于承载旧报表第一版账户维度统计。

数据来源：

- `sc.legacy.account.master`：账户类型、账户名称、账号、开户行、项目、期初余额。
- `sc.legacy.account.transaction.line`：旧库账户间往来等账户收支来源明细。
- `sc.legacy.fund.daily.line`：已承载资金日报明细中的收入、支出、当前账户余额、当前银行余额、账实差异。

行结构：

- `账户类型` 汇总行。
- `账户` 明细行。

指标结构：

- 期初余额。
- 收入金额。
- 支出金额。
- 累计收款。
- 累计支出。
- 账户往来。
- 当前账户余额。
- 当前银行余额。
- 账实差异。
- 明细数。

## 当前口径

旧过程中的 `LJSK`、`LJZC` 涉及多张历史资金往来表。第一版为了先形成可用报表入口，采用已承载资金日报明细作为可解释口径：

- `收入金额` = 已承载账户往来来源中的收入方向金额。
- `支出金额` = 已承载账户往来来源中的支出方向金额。
- `累计收款` = 已承载累计收支来源收入合计；当前缺失时回退为已匹配资金日报收入合计。
- `累计支出` = 已承载累计收支来源支出合计；当前缺失时回退为已匹配资金日报支出合计。
- `账户往来` = 收入金额 - 支出金额。
- `当前账户余额` 优先取最新资金日报当前账户余额；缺失时回退为 `期初余额 + 收入金额 - 支出金额 + 累计收款 - 累计支出`。

- 模型可通过 Odoo 模块升级创建 SQL view。
- 财务读、财务办理、财务管理能力组具备只读访问。
- 财务账款菜单出现 `账户收支统计表`。
