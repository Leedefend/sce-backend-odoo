# 账户收支统计表承载 Batch-S

## 批次定位

- Layer Target：Domain Projection
- Module：`addons/smart_construction_core`
- 旧报表：账户收支统计表
- 旧过程：`Report_SP_USP_Select_ZHSZTJB_GS_Tree`

## 已承载内容

本批新增只读投影模型 `sc.account.income.expense.summary`，用于承载旧报表第一版账户维度统计。

数据来源：

- `sc.legacy.account.master`：账户类型、账户名称、账号、开户行、项目、期初余额。
- `sc.legacy.fund.daily.line`：已承载资金日报明细中的收入、支出、当前账户余额、当前银行余额、账实差异。

行结构：

- `账户类型` 汇总行。
- `账户` 明细行。

指标结构：

- 期初余额。
- 收入金额。
- 支出金额。
- 累计收款。
- 累计支出。
- 账户往来。
- 当前账户余额。
- 当前银行余额。
- 账实差异。
- 明细数。

## 当前口径

旧过程中的 `LJSK`、`LJZC` 涉及多张历史资金往来表。第一版为了先形成可用报表入口，采用已承载资金日报明细作为可解释口径：

- `累计收款` = 已匹配资金日报收入合计。
- `累计支出` = 已匹配资金日报支出合计。
- `账户往来` = 收入金额 - 支出金额。
- `当前账户余额` 优先取最新资金日报当前账户余额；缺失时回退为 `期初余额 + 收入金额 - 支出金额`。

## 尚未完全覆盖

旧过程还引用以下来源，仍需后续逐步对账：

- 公司财务收入、支出。
- 借还款录入。
- 账户间资金往来。
- 回款登记。
- 投标退款。
- 保证金、贷款等资金模块来源。

因此 `账户收支统计表` 仍标记为 `partial`，不能标记为 ready。

## 验收口径

- 模型可通过 Odoo 模块升级创建 SQL view。
- 财务读、财务办理、财务管理能力组具备只读访问。
- 财务账款菜单出现 `账户收支统计表`。
- 模拟生产库可查询账户类型汇总行和账户明细行。
