## 判断

1. 新系统全量经营事实口径是成立的。New Only 项目不是空口径膨胀，大多数项目都有收入合同、发票、收付款、支出合同等事实。
2. 旧库常用过程口径更窄。旧过程默认输出 195 行，而新系统带旧库项目 ID 的经营事实覆盖 765 行，说明旧过程存在输出范围收敛。
3. 不能直接删除 New Only 项目。若删除或按旧过程 195 行硬过滤，会让新系统失去大量已经迁移的经营事实。
4. 仍有两个缺口需要收口：
   - 50 个新系统报表项目没有旧库项目 ID，需要追踪项目锚点来源。
   - 61 个 New Only 项目零金额/无计价文本但进入公司报表，需要追踪 `sc_ar_ap_project_summary` 的 business key 来源。

## 后续任务

- 对 50 个无旧库项目 ID 的报表项目追踪项目来源，判断是新系统新增项目、迁移锚点缺失，还是历史特殊项目。
