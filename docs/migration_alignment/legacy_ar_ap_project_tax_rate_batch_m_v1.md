# 应收应付报表（项目）抵扣比例口径修正 Batch-M

## 1. 批次定位

- Layer Target：业务事实分析层 / Domain Projection
- Module：`addons/smart_construction_core`
- Reason：Batch-K 审计发现 `SF` 抵扣比例只有 2 行非零，疑似按项目+往来单位行计算导致低覆盖。回查旧库过程后确认旧系统按项目级税额计算比例，再在每个往来单位行展示。

## 2. 旧库口径确认

旧过程 `UP_USP_SELECT_YSYFHZB_XM` 中：

- `KPDJSE` 销项税额字段按 `XMID + SPFMC=A.DWMC` 计算，是行级客户往来单位税额。
- `DKZE` 抵扣税额字段按 `XMID + KPDWID=A.ID` 计算，是行级开票单位税额。
- `SF` 抵扣比例没有往来单位过滤，只按 `XMID` 汇总：
  - 分子：项目下有效抵扣税额合计
  - 分母：项目下有效销项税额合计
  - 分母为 0 时返回 0

因此，`SF` 不是行级比例，而是项目级比例在报表行上重复展示。

## 3. 修复策略

`sc.ar.ap.project.summary` 新增 `project_tax_rate` CTE：

- 从 `output_invoice` 按项目汇总销项税额
- 从 `tax_deduction` 按项目汇总抵扣税额
- 计算 `项目抵扣税额 / 项目销项税额`
- 最终 SELECT 按 `project_id` 关联到所有报表行

保留行级 `output_tax_amount` 和 `deduction_tax_amount` 不变，只修正 `tax_deduction_rate` 的计算维度。

- 有销项税额项目：`579`
- 同时有销项税额和抵扣税额项目：`315`
- 项目销项税额合计：`177414371.67`
- 项目抵扣税额合计：`56009185.14`
- 非零抵扣比例范围：`0.002338024889657552` 到 `2.4440224394686614`

## 5. 风险与回滚

- P2：`tax_deduction_rate` 是项目级比例，用户按往来单位行查看时会看到同一项目下重复显示同一个比例。
- P2：透视表中比例字段仍不应作为简单求和指标使用，后续如需要更友好的分析，应增加项目级专用报表或禁止比例字段在透视中求和。
- 回滚：回退本批提交并升级 `smart_construction_core`。
