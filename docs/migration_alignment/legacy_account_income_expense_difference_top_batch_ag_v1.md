# 账户收支统计差异 Top 清单 Batch-AG

## 批次定位

- Layer Target：Migration Evidence / Domain Projection Validation
- Module：`scripts/migration`, `docs/migration_alignment`
- 数据库：`sc_prod_sim`
- 目标：在三层对账矩阵基础上输出账户级差异 Top 清单，并标注差异原因。

## 原因分类

- `fallback_match`：正式账户口径中，历史单据账户 ID 与账户主数据 ID 不完全一致，但通过账号/账户名称兜底绑定到正式账户。
- `supplemental_account`：历史单据引用了不存在于 `C_Base_ZHSZ` 的账户锚点，Batch-AD 已创建历史来源账户承载。

## 判断

当前账户收支统计差异已能按原因解释：

- 正式账户差异：主要来自账号/账户名兜底绑定。
- 连续经营差异：主要来自历史来源账户补齐。
- 后续业务验收可以直接围绕上述账户清单逐项确认账户归属，而不是继续怀疑报表来源缺失。
