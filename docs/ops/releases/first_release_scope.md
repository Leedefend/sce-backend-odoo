# First Release Scope

## 首发定义

- 首发产品：`施工企业项目管理系统（首发版）`
- 首发切片：`项目创建 -> 驾驶舱`
- 第二切片：`项目创建 -> 驾驶舱 -> 计划 -> 执行`

## 首发只做什么

- 用户登录进入首发前端
- 从项目创建入口完成最小字段立项
- 创建后自动进入项目驾驶舱
- 在项目驾驶舱查看首发 block 与基础导航

## 首发明确不做什么

- 不做 `成本闭环`
- 不做 `合同付款闭环`
- 不做 `结算分析`

## 不承诺什么

- 不承诺成本写侧业务闭环
- 不承诺合同/付款审批全链路上线
- 不承诺结算分析产品化交付
- 不承诺全行业场景一次性交付

## 用户路径

1. 访问登录页并完成登录
2. 进入 `项目创建`
3. 填写最小必填字段并提交
4. 系统跳转到 `项目驾驶舱`
5. 在驾驶舱查看项目状态与首发 block

## 演示路径

1. 登录
2. 进入 `项目创建`
3. 创建演示项目
4. 自动进入 `项目驾驶舱`
5. 展示驾驶舱 block、导航、基础状态信息

## 销售口径说明

- 可说：
  - “首发版已交付施工企业项目管理的核心起步链路，覆盖项目创建到驾驶舱。”
  - “第二阶段扩展到计划与执行。”
- 不可说：
  - “已经完成施工企业全业务闭环。”
  - “已经完成成本、合同、付款、结算全链路上线。”

## 对外话术模板

- 标准版：
  - “当前首发版聚焦施工企业项目管理核心起步链路，已上线项目创建到驾驶舱；计划与执行作为第二切片推进。成本、合同付款、结算分析不在本次首发承诺范围内。”
- 演示版：
  - “本次演示展示首发切片：项目创建成功后自动进入项目驾驶舱。其余业务线不在本次首发承诺范围内。”
