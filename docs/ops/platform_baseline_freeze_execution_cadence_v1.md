# 平台基线冻结与执行节奏规则 v1

## 1. 适用范围

本规范用于当前平台连续迭代链的执行节奏治理，覆盖：

- 业务事实层已封板后的能力扩展阶段
- 契约一致性与前端承接的一致化阶段
- 配置中心类低风险增量阶段

不覆盖：

- 新业务大专题立项
- 高风险权限/财务语义重构
- 跨模块架构迁移批次

## 2. 基线冻结原则

### 2.1 冻结对象

以下对象进入冻结面（仅允许增量、不允许语义倒退）：

- 原生业务事实闭环结论
- 契约冻结面与消费者依赖面
- 配置中心 acceptance-pack 主链
- `sc_test` 运行时验证基线（`http://localhost:8071`）

### 2.2 冻结规则

- 未经新任务线授权，不得修改已封板结论语义。
- 新增能力必须通过 optional/additive 方式接入。
- 不得用前端补丁替代后端事实或契约缺口修复。
- 不得将验证临时样本误写为业务真相。

## 3. 执行节奏规则

### 3.1 单轮目标约束

每一轮只允许一个主目标：

- 要么补证据
- 要么补最小能力
- 要么补治理规则

禁止同轮混入多专题扩散。

### 3.2 批次顺序

标准节奏固定为：

1. 任务合同
2. 最小改动
3. 证据更新
4. 迭代日志

若用户明确要求封板或暂停验证，可跳过验证执行，但必须记录跳过原因。

### 3.3 环境锚点

运行时证据默认锚定：

- `ENV=test`
- `DB_NAME=sc_test`
- `E2E_BASE_URL=http://localhost:8071`

偏离该锚点时，必须在报告中显式说明并给出原因。

## 4. 变更门禁规则

### 4.1 允许变更

- `docs/ops/**` 治理文档
- `agent_ops/tasks/**` 任务合同
- `agent_ops/reports/**` 迭代报告
- `agent_ops/state/task_results/**` 结果快照
- 低风险 verify 脚本增量（需任务授权）

### 4.2 禁止变更（本专题默认）

- 业务模型语义改写
- ACL / record rule 扩展
- 财务语义对象深改
- 前端权限补丁

## 5. 节奏等级定义

- `L0 封板态`：仅允许治理文档与任务切换，不执行 verify。
- `L1 证据态`：允许 verify 执行与证据补齐，不引入新功能。
- `L2 增量态`：允许低风险能力增量与验收。
- `L3 扩展态`：需要新的专题任务线授权后进入。

当前批次默认等级：`L0 -> L1` 受用户指令切换。

## 6. 报告口径规范

每轮报告必须包含：

- 本轮目标
- 变更清单
- 证据状态（执行/跳过）
- 风险结论
- 下一轮建议

当用户明确禁止 verify 时，报告必须写明：

- `Verification: Skipped by explicit user instruction`

## 7. 下一步执行建议

在本基线下，下一轮优先顺序：

1. 维持封板对象不回退
2. 按单目标节奏推进低风险专题
3. 所有新专题先建任务合同再执行
