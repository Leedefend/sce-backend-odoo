# TP-04｜项目叙事卡规则说明 v1

目标：让 PM 在项目概览区一眼看懂“项目在说什么”。
规则为 if/else 文案模板，不引入复杂算法。

## 规则清单（v1）

| 规则编号 | 触发条件 | 叙事文案 | 级别 | 备注 |
| --- | --- | --- | --- | --- |
| N-01 | 生命周期=在建 AND 成本偏差 < 0 | 项目成本已超出预算，当前存在成本压力。 | 黄 | 负值=超预算 |
| N-02 | 生命周期=在建 AND 成本偏差 >= 0 | 项目成本处于预算范围内，继续保持。 | 灰 | 正常提醒 |
| N-03 | 已结算金额 > 0 AND 已付款金额 = 0 | 项目已结算但未付款，建议关注资金安排。 | 黄 | 结算-付款 |
| N-04 | 进度完成率=0 且 无进度数据标记 | 当前暂无项目进度数据，建议补充进度信息。 | 蓝 | 轻提示 |
| N-05 | 资料完备率 < 80% | 项目资料不完整，请补齐关键资料。 | 蓝 | 阈值可调 |
| N-06 | 合同数量 = 0 | 当前项目暂无合同信息，请确认是否已完成录入。 | 蓝 | 辅助提示 |

## 说明

- 叙事卡只用于“表达”，不触发业务限制。
- 叙事卡用于“当前状态描述”，不推断责任，不代表合规判断。
- 叙事卡优先级：黄 > 蓝 > 灰（多条命中时取最高）。
- 同级别多条命中时，按对项目影响的紧急性排序展示。
- 阈值（如 80%）为展示策略，后续可配置化。
