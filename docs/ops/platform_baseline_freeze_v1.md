# 平台基线冻结 v1

## 1. 冻结结论

以下专题在当前阶段已封板（sealed）：

1. `config.item` 已封板
2. `config.role.entry` 已封板
3. `config.home.block` 已封板

以上封板结论视为当前平台基线，不得在无新专题授权下回退语义。

## 2. 门禁唯一性

当前阶段唯一门禁为：

- `acceptance-pack`

说明：

- 通过 `acceptance-pack` 视为该专题收口完成。
- 不允许以额外临时 verify 叠加替代主门禁。

## 3. 禁止事项（硬规则）

封板后默认禁止：

1. 不得新增 verify
2. 不得修改 freeze surface
3. 不得扩 acceptance-pack

如需突破以上任一规则，必须新建专题任务合同并说明新增用户价值。

## 4. 变更准入条件

仅在以下条件同时成立时允许继续变更：

- 明确新专题目标
- 明确用户可感知价值
- 明确不破坏既有封板结论

否则按冻结态处理，拒绝执行。

## 5. 下一阶段入口条件

满足本冻结文档后，才允许进入下一能力专题：

- `ITER-1356` 用户价值优先能力

示例方向：

- 首页拖拽排序（轻量）
- 快捷入口管理
- 配置中心 UI 分组
