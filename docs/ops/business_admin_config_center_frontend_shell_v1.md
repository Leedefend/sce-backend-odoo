# 业务管理员可视化配置中心 v1 · Phase 3 前端壳层设计

## 1. 设计目标

以“列表 + 表单”方式承载配置中心，不做低代码全能平台，不引入模型特判，不重写权限逻辑。

## 2. 信息架构（IA）

一级入口：`业务管理员配置中心`

二级分区：
- `系统参数配置`
- `角色入口配置`
- `首页区块配置`

每个分区统一采用：
- 列表页（检索/筛选/状态）
- 表单页（新增/编辑/发布）

## 3. 页面壳层结构

## 3.1 列表页（通用壳）

通用区域：
- 过滤栏：关键字、分类、作用域、发布状态。
- 数据表：核心字段 + 审计字段简版。
- 右上角操作：新建、导出（可选）、刷新。

最小能力：
- 搜索、排序、分页。
- 进入详情编辑。
- 按发布状态过滤（draft/active/retired）。

## 3.2 表单页（通用壳）

通用区域：
- 基本信息区（key/code/分类/作用域）。
- 配置内容区（按类型渲染输入控件）。
- 审计信息区（创建/更新/变更单）。
- 发布控制区（保存草稿、发布、停用）。

最小能力：
- 新建、编辑、保存。
- 发布前校验（字段合法性 + 白名单校验）。
- 发布后状态切换。

## 3.3 变更预览区（轻量）

仅展示影响范围说明：
- 影响角色/公司范围。
- 影响入口或区块列表。
- 不展示业务数据推演，不做流程模拟器。

## 4. 路由与入口建议

建议路径：
- `/admin/config-center`（首页）
- `/admin/config-center/system-params`
- `/admin/config-center/role-entries`
- `/admin/config-center/home-blocks`
- `/admin/config-center/:slice/:id`（详情）

入口规则：
- 仅业务管理员可见配置中心入口。
- 子菜单可按对象分区展开。

## 5. 交互约束

- 禁止模型特判：不得为某业务模型写专用前端逻辑。
- 禁止权限补丁：可见/可写最终由后端权限与 contract 判定。
- 禁止拖拽式页面引擎：当前阶段只支持结构化表单配置。
- 禁止“前端先定义语义”：所有语义以后端配置模型供给为准。

## 6. 可用性与风控要点

- 可用性：
  - 列表检索应在 2~3 次点击内完成定位。
  - 表单保存/发布路径必须明确，不隐藏关键按钮。
- 风控：
  - 发布必须具备变更单号与变更说明。
  - 高风险配置显示二次确认。

## 7. 与 Phase 4 的联动输入

输出给 Phase 4 的内容：
- 哪些字段需要进入 contract（导航显隐/排序类）。
- 哪些字段严格留在 backend-only（参数默认值类）。
- 前端真实依赖字段清单（避免后续消费漂移）。

## 8. 本阶段结论

- `PASS`（设计阶段）：前端壳层结构明确，且满足“非低代码引擎、非模型特判、非权限补丁”约束。
