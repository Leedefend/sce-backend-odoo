# A1｜项目中心 · 数据驱动表达结构说明（Draft v0.1）

## 1. 文档目的（Scope）

本文件用于统一定义「项目中心」的表达结构与角色信息密度规则，作为后续

- 页面设计
- 权限表达
- 数据摘要
- 叙事与异常提示

的最高约束文档。

本文档不讨论具体权限实现方式，只定义“应当如何表达”。

---

## 2. 核心设计原则（不可违背）

### P1. 结构优先于权限

- 所有角色使用同一套页面结构
- 差异只体现在信息密度与默认呈现方式

### P2. 表达优先于控制

- 能力不足的角色：
  - 不展示不可执行入口
  - 但尽量保留只读摘要

### P3. 数据即状态

- 页面表达由：
  - 项目数据状态
  - 项目生命周期
  - 用户角色
  共同决定

### P4. 不制造 AccessError

- 任一可见入口：
  - 要么可达
  - 要么被 UI 预先消解

---

## 3. 项目中心的四大表达分区（固定结构）

下列四区为强约束结构，所有角色一致，仅密度不同。

---

### 3.1 项目概览区（Overview）

**定位**：

> “这个项目现在怎么样？”

**内容性质**：

- 汇总
- 判断
- 叙事

**包含信息类型**：

- 项目基本信息（名称、类型、阶段）
- 项目生命周期状态
- 核心指标摘要
- 项目叙事卡（Narrative Card）

**角色密度规则**：

| 角色 | 表达密度 |
| --- | --- |
| 项目经理 | 默认展开，重点区域 |
| 经办 | 展示摘要，可跳转 |
| 管理员 | 展示全部，含配置入口 |

---

### 3.2 业务执行区（Execution）

**定位**：

> “项目是怎么被干出来的？”

**内容性质**：

- 结构
- 过程
- 执行

**包含模块**：

- WBS / 工程结构
- BOQ / 工程量清单
- 任务/施工拆解

**角色密度规则**：

| 角色 | 表达策略 |
| --- | --- |
| 项目经理 | 只读结构 + 汇总指标 |
| 经办 | 明细可操作 |
| 管理员 | 结构 + 维护能力 |

**约束**：

- 若模块无数据，且无权限，展示只读摘要或提示
- 不显示不可执行入口

---

### 3.3 管理支撑区（Management）

**定位**：

> “项目是否在受控范围内？”

**内容性质**：

- 对比
- 偏差
- 风险

**包含模块**：

- 合同
- 预算 / 成本
- 进度计量
- 财务相关指标（只读）

**角色密度规则**：

| 角色 | 表达策略 |
| --- | --- |
| 项目经理 | 汇总指标 + 趋势 |
| 经办 | 明细 + 操作 |
| 管理员 | 明细 + 配置 |

**约束**：

- 项目经理不出现配置型动作
- 但必须能看到结果与偏差

---

### 3.4 协作与系统区（Collaboration / System）

**定位**：

> “项目的协作与系统属性”

**内容性质**：

- 辅助
- 系统
- 配置

**包含模块**：

- 文档
- 协作人员
- 系统字段（分析账户等）

**角色密度规则**：

| 角色 | 表达策略 |
| --- | --- |
| 项目经理 | 只读 |
| 经办 | 可维护 |
| 管理员 | 全能力 |

**约束**：

- 此区永远位于所有业务页签之后

---

## 4. 角色 × 表达密度总表（关键）

| 角色 | 页面结构 | 信息密度 | 配置能力 |
| --- | --- | --- | --- |
| 项目经理 | 完整 | 低～中 | 无 |
| 经办 | 完整 | 中～高 | 有限 |
| 管理员 | 完整 | 高 | 全部 |

**禁止**：为角色定制完全不同页面。

---

## 5. 生命周期驱动的表达规则（基础版）

| 生命周期 | 默认展开 | 默认折叠 |
| --- | --- | --- |
| 立项 / 在建 | 执行区 | 协作区 |
| 竣工 | 管理支撑区 | 执行区 |
| 质保 | 协作区 | 执行区 |

---

## 6. 本文档的约束效力

- 新增入口 / 页签：
  - 必须声明所属分区
- 新增字段 / 指标：
  - 必须声明默认角色密度
- 新增权限：
  - 不得破坏本结构一致性

---

## 7. 当前阶段完成定义（A1 Done）

当满足以下条件，即认为 A1 完成：

1. 项目中心页面可完整映射至四大分区
2. demo_pm 与 admin 页面结构一致，仅密度不同
3. 任一入口都能在本文件中找到表达归属解释
