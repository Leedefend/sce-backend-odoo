# 能力阶段定义

本文定义项目的能力阶段划分。

## 总览
- P0：基线稳定与生产安全。
- P0.1：运维治理与文档对齐。
- P1：功能扩展。
- P2：企业级深度与规模能力。

## P0
- 定位：生产安全的稳定基线。
- 关键词：guard、release、baseline。
- 承诺：
  - 生产 guard 落地。
  - Release checklist 完整。
  - base profile 可用。
- 不承诺：
  - 基线之外的功能扩展。

## P0.1
- 定位：运维一致性与治理。
- 关键词：seed 生命周期、边界、入门。
- 承诺：
  - Seed 生命周期文档与 guard。
  - 模块边界制度化。
  - 开发者入口明确。
- 不承诺：
  - 新业务能力。

## P1
- 定位：功能增长。
- 关键词：工作台深化、业务流、流程。
- 承诺：
  - 业务流扩展。
  - 面向用户的功能增强。
- 不承诺：
  - 企业级规模能力。

## P2
- 定位：企业成熟度。
- 关键词：合规、审计、规模。
- 承诺：
  - 高级治理与可审计性。
  - 企业级扩展能力。
- 不承诺：
  - 实验性流程。

## 版本关系
- v0.3.0-stable 对应 P0 封板。
- P0.1 在不破坏 P0 承诺的前提下演进。
