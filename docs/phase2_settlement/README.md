# Phase-2 收官说明（结算中心·核心版）

## 1. 本阶段解决的问题
- 建立结算额度闭环：结算单（额度载体）→ 付款/收款申请（额度消耗）→ 审批 → 资金台账（事实记录）。
- 硬校验防超付：剩余额度 ≤ 0 或申请金额超额时阻断提交/完成，并在 UI 前置风险提示。
- 可审计：资金台账只读展示项目/往来/结算单/付款申请/方向/金额/状态，为核对与审计提供事实视图。

## 2. 当前边界（已做 / 未做）
- ✅ 已做：结算单额度摘要（总额/已付/剩余，风险红色）；付款申请结算依据区信息前置+红色提示；台账只读列表/表单与菜单；模型/视图/权限一致并通过升级。
- ✅ 已做：幂等入账、防重复付款、额度耗尽自动完成结算单（剩余≤0）。
- ⏸ 未做：三单匹配（合同/采购/发票）；超额审批策略化；会计凭证/银行接口；复杂金额拆分。

## 3. 下一阶段起点（推荐顺序）
- Phase-3.0：三单匹配基础版（合同↔结算单，采购订单↔结算单，发票占位），先做齐全性/一致性校验，不做复杂拆分。
- Phase-3.1：超额付款策略化（基于现有额度不足 flag，增加策略参数 + Tier 升级流）。
- Phase-4：会计凭证/资金侧集成，放在最后处理。
