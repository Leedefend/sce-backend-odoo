# Product Delivery

This directory contains productized delivery definitions and handoff materials.

## Versioned Delivery Packages
- `v1/`: Phase A delivery package (pilot-ready baseline)

## Rules
- Keep all delivery-scoped artifacts under a version folder (`v1`, `v2`, ...).
- Any scope change must update version changelog and mapping artifacts.
- Delivery docs are audience-facing (implementation, presales, and operations).
