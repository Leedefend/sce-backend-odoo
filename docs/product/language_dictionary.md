# 产品语言法典（Phase 1）

> 目标：统一用户可见的关键词汇，避免同义词混用。

## 按钮词

- 保存：默认不在业务表单显式出现（系统行为）
- 暂存：草稿态主要动作（等价于保存草稿）
- 提交：进入流程/审批/立项的动作
- 提交立项：项目从草稿进入在建
- 放弃变更：编辑器语义，业务表单草稿态尽量隐藏
- 取消：终止当前动作/流程
- 确认：完成关键动作或进入下一阶段

## 生命周期词（项目）

- draft = 草稿
- in_progress = 在建
- paused = 停工
- done = 竣工
- closing = 结算中
- warranty = 保修期
- closed = 关闭

## 页面词

- 台账：以业务对象为维度的列表/汇总入口
- 执行表：记录执行过程与进度的页面
- 汇总：以项目/合同维度汇总视图
- 明细：单据或行级明细
- 导出：导出报表或清单

## 一致性规则

1. 同一语义在全站只允许出现一个词（禁止“保存/暂存”混用）。
2. 业务入口优先使用“台账/执行表”等业务语义，不使用系统术语。
3. 草稿态优先“暂存/提交”，避免“放弃变更”。
