## 统计口径
- `P0`：阻断首发或核心路径不可用，必须在首发前清零。
- `P1`：高风险问题，允许存在但需有明确缓解与上线后跟踪策略。
- `P2`：一般问题，不阻断首发。

## 闭环规则
- 每条问题必须包含“复现步骤 + 场景 + 角色 + 严重级别”。
- 每日更新状态与关闭时间。
- 首发评审时必须输出按严重级别汇总结果。
